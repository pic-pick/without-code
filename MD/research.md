# 코드없이 (withoutcode) — 서비스 분석 보고서

> PRD(`MD/without-code.md`) 기반 심층 분석. 작성일: 2026-03-22

---

## 1. 서비스 정의

### 핵심 명제
**"비개발자를 위한 바이브코딩 쇼케이스 SNS"**

AI 툴(Cursor, Lovable 등)로 앱·서비스를 만든 비개발자가 결과물을 공유하고, 서로에게 배우는 한국 최초 특화 커뮤니티 플랫폼.

### 포지셔닝
| 비교 축 | 코드없이 | 기존 대안 |
|--------|---------|---------|
| 타겟 | 비개발자 | GitHub(개발자), ProductHunt(영어권) |
| 언어 | 한국어 특화 | 영어 중심 |
| 핵심 콘텐츠 | AI 결과물 + 프롬프트 공개 | 코드, 제품 자체 |
| UX 톤 | "만든 것" (비개발자 언어) | 개발자 문화·용어 |

## 2. 브랜드 시스템 분석

### 컬러 팔레트 구조
| 변수명 | 헥스 | 용도 | 중요도 |
|--------|------|------|--------|
| Ink (Primary) | `#1C1C1A` | 메인 배경·텍스트 | 필수 |
| Signature (Red) | `#E8483A` | 로고·CTA·포인트 액센트 | **로고 전용** — 남용 금지 |
| White | `#F7F7F5` | 라이트 배경·역전 텍스트 | 필수 |
| Charcoal | `#444441` | 보조 텍스트 | 필수 |
| Stone | `#888780` | 서브텍스트·설명 | 필수 |
| Cloud | `#D3D1C7` | 구분선·보더 | 필수 |
| Light BG | `#F2F0EB` | 카드 배경(라이트) | 필수 |

**핵심 규칙**: 레드(`#E8483A`)는 로고와 CTA 버튼에만. 임의 컬러 추가 불가.

### 타이포그래피 규칙
- **한글**: Pretendard (웹폰트, 무료 오픈소스)
- **영문**: Inter
- **코드/모노**: Courier New (로고 `</>` 전용)
- **Fallback**: `Apple SD Gothic Neo, Malgun Gothic, sans-serif`

### 로고 의미론
`</>` — 코드 태그를 취소선(레드)으로 그어서 "코드 없이"를 시각적으로 표현. 모노스페이스 폰트 사용. SVG로 정의되어 있으며 둥근 모서리의 `200×200` 아이콘 형태.

### 다크/라이트 혼합 디자인
MVP에서는 다크모드를 별도 지원하지 않음. 다크(`#1C1C1A`)와 라이트(`#F7F7F5`) 혼합 레이아웃은 **의도적 브랜드 선택** — 규칙에 따라 그대로 유지해야 함.

## 3. 기능 명세 심층 분석

### Phase 1 — 기반 구축 (최우선)

#### 소셜 로그인
- 카카오 / 구글 / GitHub 세 가지
- 카카오가 한국 타겟 핵심 채널
- GitHub 로그인은 후속 GitHub 연동 기능과 자연 연결 포인트

#### 프로젝트 쇼케이스 업로드
필드 구성:
| 필드 | 타입 | 제약 |
|------|------|------|
| 제목 | text | - |
| 설명 | text/markdown | - |
| 데모 링크 | url | `https://` 필수 검증 |
| GitHub 링크 | url | 선택 |
| 툴 태그 | tag[] | 다중 선택 |
| 썸네일 이미지 | image | Supabase Storage `thumbnails` 버킷, 최대 5MB |

**`demo_url` 필수화** — 스팸·저품질 콘텐츠 방어를 위한 진입장벽으로 활용.

#### 메인 피드
- 최신순 / 인기순 (추후 알고리즘화 예정)
- **자유 게시글 + 프로젝트 썸네일 박스** 혼합 구성
- 인스타그램형 그리드 카드 + 일반 텍스트 포스트가 공존하는 하이브리드 피드

#### 프로젝트 상세 페이지
- 썸네일, 설명, 데모 링크, 툴 태그, 프롬프트(선택 공개) 표시

### Phase 2 — 소셜 기능 (3~4주)

#### 소셜 상호작용
- 좋아요 / 댓글 / 북마크 — 기본 SNS 인터랙션 3종 세트
- 팔로우 — 크리에이터 팔로잉 그래프 형성

#### 유저 프로필
- 포트폴리오 역할: 업로드한 프로젝트 모아보기
- 팔로워/팔로잉 수

#### 검색·필터
- 툴 태그 기반 필터 (예: Cursor로 만든 것만 보기)
- 키워드 검색

#### 알림
- 댓글 알림, 좋아요 알림

---

### Phase 3 — 성장 기능 (5~8주)

#### 주간 베스트
- 큐레이션/알고리즘으로 우수 프로젝트 선정 → 바이럴 동기 제공

#### OG 이미지 자동 생성
- SNS 공유 최적화 → 외부 유입 루프 강화

#### 프롬프트 공개 기능 ⭐ 핵심 차별화
- 프로젝트 제작 시 사용한 AI 프롬프트를 선택적으로 공개
- 비개발자에게 가장 가치 있는 정보 ("어떻게 만들었는지")
- 지식 공유 + 학습 커뮤니티 성격 강화

#### Q&A 게시판
- 바이브코딩 방법·툴 사용법 질의응답

## 5. 수익 모델 로드맵

```
단기 ──────────────────────────────────────────── 장기
  │                      │                         │
  ▼                      ▼                         ▼
프리미엄 멤버십       AI 툴사 광고·스폰서십      바이브코더↔개발자 매칭
(포트폴리오 강조,    (Cursor, Lovable 등        (프로젝트 고도화 의뢰,
 고급 통계,          타겟 광고)                  매칭 수수료)
 프로필 커스텀)
```

**주목할 점**: Cursor, Lovable 등 AI 툴사가 광고주 — 플랫폼 자체가 이 툴들의 파워유저 집합소이므로 광고 효율이 높음. 양면 시장(바이브코더 + 개발자)으로의 진화가 장기 핵심 가치.

## 6. 콜드스타트 전략 분석

| 전략 | 방법 | 예상 효과 |
|------|------|----------|
| 시드 유저 직접 섭외 | 바이브코딩 유튜버·인플루언서 5~10명 | 초기 콘텐츠 밀도 확보 |
| 커뮤니티 침투 | 오픈카톡·디스코드·커리어리 | 얼리어답터 모집 |
| 콘텐츠 마케팅 | 네이버 블로그·브런치 | SEO + 공감 콘텐츠 유입 |
| 빌드 인 퍼블릭 | 서비스 자체를 바이브코딩으로 제작·공개 | **마케팅 = 제품 자체** |

**"코드없이로 코드없이를 만들었습니다"** — 이 스토리는 단순한 홍보를 넘어 서비스 정체성을 직접 증명하는 최강의 마케팅 내러티브.

## 7. 차별화 포인트 평가

### 1. 프롬프트 공개 기능
가장 강력한 차별점. GitHub의 "코드 공개"와 같은 문화를 비개발자 세계에 이식. "결과물"이 아닌 "과정"을 공유하는 플랫폼으로 포지셔닝.

### 2. 비개발자 전용 UX
용어 장벽 제거. "commit", "deploy", "repo" 대신 비개발자 언어 사용. 타겟 정밀도 높음.

### 3. 한국어 특화
국내 유일 포지션. 바이브코딩 커뮤니티가 영어 중심인 상황에서 한국어 허브로 선점 효과.

### 4. 툴 태그 생태계
Cursor, Lovable, Bolt, Claude 등 툴별 커뮤니티가 자연 형성됨 → 툴사와의 파트너십·광고 수익 연결 고리.

### 5. GitHub 연동 (로드맵)
바이브코딩 결과물이 실제 코드 레포와 연결 → 개발자 생태계 브릿지.

## 8. 리스크 및 대응 분석

### 콜드스타트 (HIGH)
- **위험**: 초기 콘텐츠 없으면 유저 이탈
- **대응**: 시드 유저 10명 확보 + 직접 콘텐츠 시딩
- **평가**: 인플루언서 파트너십 전략이 현실적. 바이브코딩 유튜버 채널 시청자층과 타겟이 정확히 일치.

### 바이브코딩 트렌드 의존성 (MEDIUM)
- **위험**: 바이브코딩 트렌드 소멸 시 플랫폼 가치 감소
- **대응**: "AI 활용 창작 커뮤니티"로 포지션 확장 가능
- **평가**: AI 도구 기반 창작은 수년간 성장 트렌드. 단기 리스크 낮음.

### 대형 플랫폼 진입 (MEDIUM)
- **위험**: 네이버·카카오 등이 유사 기능 출시
- **대응**: 비개발자 특화 UX + 한국어 선점 효과
- **평가**: 커뮤니티는 스위칭 비용이 높음. 초기 커뮤니티 문화 형성이 핵심.

### 스팸/저품질 콘텐츠 (LOW)
- **위험**: 실제 결과물 없이 홍보성 게시물 도배
- **대응**: 데모 링크(`https://`) 필수화
- **평가**: 간단하지만 효과적인 필터. 실제로 배포된 결과물만 업로드 가능.

## 10. 종합 평가

### 강점 요약
1. **시장 타이밍** — 바이브코딩 트렌드 초입에서 한국어 특화 선점
2. **명확한 타겟** — 비개발자 AI 창작자, 현재 갈 곳 없음
3. **차별화 기능** — 프롬프트 공개는 경쟁 플랫폼 없음
4. **수익 경로 명확** — 멤버십 → 광고 → 매칭의 단계적 수익화
5. **마케팅 내러티브 강함** — "코드없이로 코드없이를 만들다"

### 주요 실행 포인트
1. **Phase 1 완성도** 최우선 — 소셜 로그인 + 업로드 + 피드 + 상세 페이지
2. **Supabase RLS** 반드시 설정 — 보안이 신뢰의 기반
3. **모바일 퍼스트** 설계 철저히
4. **시드 유저 확보**를 개발 병행 진행 — 런칭 시 콘텐츠 공백 방지
5. **프롬프트 공개 기능** Phase 3에 배치되어 있으나 차별화 핵심이므로 설계 초기부터 DB 스키마에 포함 권장

### 우선순위 개발 순서
```
소셜 로그인 (카카오 우선)
  → 프로젝트 업로드 (데모링크 검증 + 이미지 업로드)
    → 메인 피드 (최신순)
      → 상세 페이지
        → 좋아요/댓글/북마크
          → 프로필 + 팔로우
            → 검색/필터
              → 프롬프트 공개 기능
```
