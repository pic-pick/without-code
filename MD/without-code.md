## 3. 핵심 기능 명세

> 전체 기능 현황: **15개 카테고리 / 87개 세부 기능 / 14개 플로우차트** (상세 명세: `Doc/FeatureDetail-FlowChart.html`)
> 우선순위 기준 — MUST: 반드시 / SHOULD: 강력 권장 / COULD: 있으면 좋음

### Phase 1 — 기반 구축 ← 최우선

#### 인증 & 계정
- [ ] 소셜 로그인 (카카오 / 구글 / GitHub) — OAuth 2.0, 최초 로그인 시 자동 회원가입
- [ ] 이용약관 / 개인정보 동의 (회원가입 시 필수, 마케팅 수신 선택)
- [ ] JWT 세션 관리 (Refresh Token 자동 갱신, 다중 디바이스)
- [ ] 계정 탈퇴 — 7일 유예기간 후 완전 삭제, GDPR 준수
- [ ] 비로그인 열람 허용 (피드·상세 읽기 전용)

#### 온보딩
- [ ] 신규 가입 후 닉네임·프로필·관심 툴 태그(3개 이상) 설정
- [ ] 서비스 소개 슬라이드 3장 (최초 1회)

#### 프로젝트 쇼케이스 업로드
- [ ] 제목(60자) / 설명(마크다운 지원, 500자) / 데모 URL(https:// 필수) / GitHub URL(선택)
- [ ] 썸네일 이미지 (5MB 이하, Supabase Storage `thumbnails`) + **추가 스크린샷 최대 10장**
- [ ] GIF / 동영상 데모 (30초 이하)
- [ ] 툴 태그 선택 (최대 5개) + **카테고리 선택** (웹앱 / 모바일앱 / 자동화 / 디자인 / 게임 등)
- [ ] 초안 자동저장 (30초 간격), 이어 쓰기
- [ ] 업로드 % 프로그레스바, 수정 시 "수정됨" 뱃지

#### 메인 피드
- [ ] 통합 피드 (프로젝트 쇼케이스 카드 + 자유 게시글 혼합)
- [ ] 정렬: 최신순 / 인기순 (좋아요+북마크 가중치)
- [ ] 무한 스크롤 (cursor-based pagination, 스켈레톤 로딩)
- [ ] 풀다운 새로고침 (모바일)
- [ ] 자유 게시글 작성 (텍스트 최대 1,000자, 게시글 수정·삭제)

#### 프로젝트 상세 페이지
- [ ] 이미지 갤러리 / 설명 / 데모 링크 버튼 / GitHub 링크 버튼
- [ ] 제작자 정보 / 사용 툴 태그 / 조회수+좋아요+북마크 통계
- [ ] **데모 링크 OG 메타 프리뷰** (링크 카드 자동 표시)

#### 설정 기본
- [ ] 로그아웃 / 앱 버전 정보 / 이용약관·개인정보처리방침

### Phase 2 — 소셜 기능

#### 소셜 인터랙션

## 11. 콜드스타트 전략

1. **시드 유저 직접 섭외** — 바이브코딩 유튜버·인플루언서 5~10명 파트너십
2. **커뮤니티 침투** — 국내 바이브코딩 오픈카톡·디스코드·커리어리 얼리어답터 모집
3. **콘텐츠 마케팅** — "비개발자가 AI로 앱 만든 후기" 네이버 블로그·브런치 배포
4. **빌드 인 퍼블릭** — 서비스 자체를 바이브 코딩으로 제작하며 과정 공유 ("코드없이로 코드없이를 만들었습니다" 스토리가 곧 마케팅)

## 13. 차별화 포인트

1. **프롬프트 공개 기능** — 결과물뿐 아니라 "어떻게 만들었는지" 공유. 비개발자에게 가장 궁금한 정보
2. **비개발자 전용 UX** — 개발자 문화·용어 배제. "코드" 대신 "만든 것" 언어 사용
3. **한국어 특화** — 국내 유일 바이브코딩 한국어 커뮤니티
4. **툴 태그 생태계** — 어떤 AI 툴로 만들었는지 태깅 → 툴별 커뮤니티 자연 형성
4. **깃허브 연동** — 바이브 코딩 프로젝트를 자연스럽게 연결시켜, 커뮤니티과의 연동성 구현

## 15. Claude Code 작업 시 주의사항

- 모든 텍스트는 한국어 기본 (`ko` locale), 영문 지원은 Post-MVP 이후
- 컬러는 위 팔레트 외 임의 추가 금지. 레드(`#E8483A`)는 로고·CTA에만 사용
- 폰트는 반드시 Pretendard 사용 (시스템 폰트 fallback: `Apple SD Gothic Neo, Malgun Gothic, sans-serif`)
- Supabase RLS 정책 반드시 활성화 — 인증 없이 쓰기 불가
- 이미지 업로드는 Supabase Storage `thumbnails` 버킷 사용, 최대 5MB (추가 스크린샷 동일 버킷)
- `demo_url`은 반드시 `https://` 프로토콜 검증
- GitHub URL은 `github.com` 도메인 검증
- 모바일 퍼스트 설계 — 타겟 유저 대부분이 모바일로 접근
- 다크모드 미지원 (MVP 단계). 라이트/다크 혼합 디자인은 의도적 브랜드 선택
- 삭제는 항상 soft delete (프로젝트 30일, 계정 7일 유예기간 후 하드 삭제)
- 마크다운 에디터 필요 구간: 프로젝트 설명, Q&A 질문·답변 (코드 블록 하이라이팅 포함)
- 프롬프트 공개 기능은 Phase 3 — 그 전에는 DB 컬럼만 준비, UI는 미노출
- 알림 시스템: 인앱(Supabase Realtime) + 푸시(FCM) + 이메일 큐 3-track 구조
- 결제 연동: 토스페이먼츠 / 카카오페이 (전자상거래법 준수, 환불 정책 필수)
- SEO: 모든 공개 페이지에 title·description·og·twitter:card 메타 태그 자동 설정
- 비로그인 유저: 피드·프로젝트 상세 읽기 허용, 좋아요·댓글·업로드 등 쓰기는 로그인 유도 바텀시트 표시
